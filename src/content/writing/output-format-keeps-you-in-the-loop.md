我读 Anthropic 那篇讲 Claude Code 输出格式的文章时，第一反应不是「HTML 还是 Markdown」，而是另一件更要紧的事：

**agent 时代，输出格式会决定人还能不能留在判断回路里。**

它真正戳中的问题是：agent 越能干，人越容易被一大坨「看似完整」的输出淹没。最后不是 AI 写不出来，而是**你没法继续判断、比较、编辑和交接**。

Markdown 仍然简单、便携、好编辑。但当 agent 开始生成更长的规格、计划、研究报告、PR 说明，纯文本会越来越难读——说实话，超过 100 行的 Markdown，我自己也很难真正读完。

HTML 的优势不只是「更漂亮」。它能承载表格、CSS、SVG、交互控件、canvas、图片，也能做「copy as prompt」「copy as JSON」这种把你带回工作流的双向操作。

换句话说，好的 agent output 不只是让你看懂，而是**把你带回工作流**。

放进个人操作系统的视角里，我现在是这么分载体的：

- **Markdown** 适合记录；
- **`CLAUDE.md`** 适合规则；
- **Skill** 适合流程；
- **HTML** 适合高密度判断界面。

不是谁替代谁，而是给不同的工作选不同的载体。这件事我在 [资料库里整理的几篇 Claude Code 资料](/library/claude-code-best-practices) 里反复看到同一个信号。

---

所以我觉得 agent 时代的关键能力，不只是生成内容，而是**设计让人保持判断的界面**。

模型会越来越能干。能干到一定程度，你最该守住的，反而是「我还看得懂、还能改、还能交接」这条线。
