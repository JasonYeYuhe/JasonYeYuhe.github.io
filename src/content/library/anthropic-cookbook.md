## 这是什么

一个 GitHub repo，里面是一篇篇 Jupyter notebook，对应「怎么用 Claude API 做某件具体的事」——文档摘要、工具调用、子代理编排、PDF 处理、视觉问答、嵌入向量检索等。

## 怎么用

读 [Building effective agents](/library/building-effective-agents) 这类综述时，理论上你知道了「链式 / 并行 / 路由 / 评估-修正」这些 pattern 该怎么组装。但真的上手之前，最快路径是：

1. 在 Cookbook 里找最接近你场景的 notebook
2. 跑通它
3. 把它改成你的输入

比起从零开始读 API 文档拼代码，省一个数量级时间。

## 我的注

> 把 Cookbook 当作「能跑的脚手架」，不是「最佳实践模板」。它教你怎么调 API，不教你产品该不该这么做——那是 Founder OS 那一层的判断。
