## 这是什么

Anthropic 2026 年 5 月发布的官方 playbook，36 页，把 AI-native 创业拆成四个阶段（Idea / MVP / Launch / Scale），并在每个阶段给出目标、退出标准、典型失败模式，以及 Claude、Claude Code、Claude Cowork 三种产品形态在该阶段的具体用法。

我做了一份**非官方中文全译版**，按原版 1:1 重排（保留所有插图、配色、章节标签、表格栅格），放在 GitHub repo 里。本页只放结构摘要与我的注解；下载请走下方 GitHub 链接。

## 为什么值得读

不是「AI 提效手册」。它真正讲的是：当 AI 把「能不能做」变成默认能做之后，**创始人的稀缺能力转向了判断、验证、范围与上下文管理**——这套手册把这种转移写成了可执行的四阶段地图。

## 四阶段速览（我的注解版）

### Idea｜创意阶段
- 主要工作：研究、客户探索、对抗性验证，**不要急着写代码**。
- 退出标准：找到 problem-solution fit。能对三个问题答「是」：问题是否真实具体 / 方案是否针对真实问题 / 信号是否足以支撑构建。
- 我的注：原型不是验证，对话才是。AI 让你写代码变快，也让你**把原型误当结论**变快。

### MVP｜MVP 阶段
- 主要工作：把已验证假设变成真实用户会用的最小产品；同时把架构与上下文写下来。
- 退出标准：真实的 PMF 证据——留存、付费、推荐。
- 我的注：AI 时代的技术债是**复利的**，因为没有规格的会话每次都会重导出基础决策。`CLAUDE.md` 不是可选项。

### Launch｜发布阶段
- 主要工作：把早期牵引转成可重复、由渠道驱动的增长；硬化基础设施；建立无需创始人触发的流程。
- 退出标准：可预测获取 + 生产级可靠性 + 运营无创始人瓶颈。
- 我的注：把「自己亲手做的工作」翻译成「设计做这件工作的系统」，这是大多数创业者卡住的地方。

### Scale｜规模化阶段
- 主要工作：从产品工程转向公司工程——治理、合规、GTM、组织职能。
- 退出标准：可持续盈利 / IPO-ready / 被并购，三选一。
- 我的注：护城河来自**累积深度**——领域专长、集成深度、用户工作流锁定。这些都不是新功能能换来的。
